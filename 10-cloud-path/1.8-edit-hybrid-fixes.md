# Lesson 1.8: Fixing and Holding Identity with Edit and Hybrid Models

Image-to-image plus a good base set gets you most of the way. This lesson is about the last stretch: what to do when the scene comes out great but the face slips a little. Instead of throwing the whole image away and rerolling, you repair just the part that went wrong. This is where edit and hybrid models (Lesson 1.1) earn their place.

## The mindset shift: repair, don't reroll

Beginners delete a near-miss and generate again, hoping the dice land better. That wastes a good scene and invites fresh drift. The professional move is the opposite: when ninety percent of an image is right, **keep the ninety percent and fix the ten.** Edit and hybrid models let you do exactly that.

## The techniques

**Face fixing (inpainting).** Mask just the face and regenerate only that region, guided by your reference, while the rest of the image stays untouched. The great scene you already have is preserved, and only the identity gets corrected. This is the workhorse repair.

**Identity transfer features.** Many edit and hybrid tools have a dedicated "face reference" or "character reference" slot. You hand it your base image as the identity and a scene as the target, and it does the carrying for you. When a tool offers this, use it, it is built for this exact problem.

**Iterative refinement in a hybrid.** This is the smooth loop we have been pointing at since Lesson 1.5. Generate or image-to-image a scene, feed the result straight back into the same tool, nudge the face back on-model, and repeat in small steps. One fix at a time (Lesson 1.5, habit 1), staying in one window so you never lose your thread.

## The full cloud workflow, end to end

Here is everything in Module 1 assembled into one repeatable loop. This is the cloud path in practice:

1. **Invent your character.** Use text-to-image to generate a strong starting character, or land a good one with a tight identity block (Lessons 1.3 and 1.4).
2. **Build a small base set.** Bootstrap a clean set of angles and expressions from your best keeper (Lesson 1.7).
3. **Make the shot.** For each new scene, run image-to-image from the base image closest to what you need (Lessons 1.6 and 1.7).
4. **Repair the slips.** Fix any drift with a face-fix or identity-transfer pass in an edit or hybrid model (this lesson).
5. **Select and feed back.** Keep the best result, and add it back into your base set so the set keeps getting stronger.

Run that loop and you can hold a character, recognizably, across a whole series of scenes using cloud tools alone.

## The honest limit

Even stacked together, these are still repairs and nudges. You can reach genuinely good, consistent results this way, and for a lot of work that is the finish line. But notice what it costs: it is hands-on, every single time. There is no point where the character is simply *locked* and you stop fussing. That ceiling is real, and the next lesson names it plainly.
