# Lesson 1.7: Building a Small Base Set of Your Character

Lesson 1.6 left us with a clear weakness: a single reference image is one angle, one expression, one light, so the moment you need your character to look up, or smile, or stand in different lighting, the model is back to guessing and the face drifts. This lesson fixes that the cloud way: stop leaning on one image, and build a small set.

## Your first training dataset (the quick version)

That small set has a name you will hear constantly from here on: a **training dataset.** For now, keep the definition simple: *a small, curated collection of images of one character that you reuse as source material.* In this tier we use it as a pool of references to pull from. In the local path, the same idea gets far more powerful (you use a dataset to actually teach a model), and we go deep on it there. Same name, bigger job later.

For the cloud path, think of it as your character's reference folder.

## Building one from nothing

You do not need a photoshoot. You bootstrap the set from a single good image:

1. **Start with your best keeper.** Take the strongest on-model image you have, the one that most looks like your character (from your batch-and-select habit in Lesson 1.5).
2. **Generate variations.** Use image-to-image to produce that same character at different angles, expressions, and lighting, one change at a time, holding identity as well as you can.
3. **Select ruthlessly.** Keep only the variations that genuinely still read as the same person. Throw the rest away without mercy.
4. **Repeat until you have coverage.** A handful of solid images spanning a few angles and expressions is plenty to start.

You are slowly turning one good image into a small, trustworthy set.

## How the set earns its keep

Here is the payoff. When you need a new shot, do not always reach for the same single reference. Instead, **pick the image in your set that is closest to the shot you want**, the nearest angle, the nearest expression, and run image-to-image from *that* one.

Why this works: it shrinks the distance the model has to travel. In Lesson 1.6 we saw that drift grows the further image-to-image has to wander from its source. Matching your source to your target keeps that distance small, so the face holds far better. A profile shot comes out cleaner when you start from a profile in your set, not from a head-on portrait.

## Curate like it matters, because it does

One rule above all: **only keep images that are truly on-model.** A bad image in your set is worse than useless, because it quietly invites you to start from the wrong source and teaches drift back into your work. A small, clean set beats a large, sloppy one every time. Quality over quantity, always.

## The honest limit

This is still image-to-image underneath, so every limit from Lesson 1.6 still applies. A base set widens your coverage and cuts drift a lot, but it is still selection and nudging, not a true lock. You are giving the model better starting points, not teaching it who your character is. That deeper step, and the real craft of building datasets, lives in the local path.
