# Lesson 1.6: Image-to-Image, the Core Move

This is the lesson the whole cloud path has been pointing at. Every thread in Module 0 ended on the same instruction: stop describing the face, find a way to feed it in (Lesson 0.5). Image-to-image is the first tool that actually does that, and it is the single most important move in this tier.

Let's build it up from the thing you already know.

## Text-to-image, a quick recap

You met this in Lessons 0.1 and 1.1. In text-to-image, the only thing going in is your words. The model starts from a field of pure random noise and carves an image out of it, steered by your prompt. Because the sole input is text, and text cannot carry a face (Lesson 0.5), you get someone who *matches the words*, and a different someone every run.

## So what is image-to-image?

Image-to-image changes the starting point. Instead of beginning from pure random noise, the model begins from **your image**. It adds some noise on top of the picture you hand it, then denoises back toward something that fits your prompt. Because it started from your image, the result keeps a great deal of your image: the composition, the shapes, and most importantly the face.

The whole difference, in one line:

> **Text-to-image starts from noise. Image-to-image starts from your image.**

That mechanical change is everything, because now actual visual information is going *in*, not a description of it. This is the bridge Lesson 0.5 promised, finally made real.

## The one dial that decides everything: strength

Image-to-image has a control usually called **strength** (sometimes "denoise" or "image weight"). It sets how much noise gets added to your starting image, which is really a measure of how far the model is allowed to wander from it.

- **Low strength:** very little noise added. The model stays glued to your input. Identity holds tightly, but it also cannot change much, so it struggles to put your character in a genuinely new scene. You get something close to a copy.
- **High strength:** a lot of noise added. The model is free to make big changes, a new setting, a new pose, but with that freedom it starts re-inventing the face, and the drift comes right back.

This tension is the heart of image-to-image: **hold the identity, or change the scene. The dial trades one against the other.** The craft is finding the sweet spot for each shot, and it is fiddly, per image, by feel.

> **[FILL LATER: strength slider demo]** The same input image run at low, medium, and high strength, side by side. Identity held but scene barely changed at low, scene changed but face drifted at high. Make the trade-off impossible to miss.

## Why this is a solution

Compared with pure prompting, image-to-image is a giant step up. You go from "a stranger every time" to "usually a close, recognizable version of the same person." You are finally handing the model the face instead of hoping your adjectives rebuild it. For a great deal of cloud work, that is the difference between unusable and genuinely good.

## Why it is not a perfect one

Be honest about the limits, because you *will* hit them:

- **The strength trade-off never fully goes away.** Big changes to the scene keep costing you identity. There is no setting that gives you total freedom and a perfectly locked face at the same time.
- **One reference is one angle, one expression, one light.** The model still has to guess the rest of the face. Turn the head, change the expression, and it is guessing again, so drift sneaks back in.
- **It drags along what you did not ask for.** Pose, framing, lighting, and background from the reference tend to come with the face.
- **It is still the same memoryless engine** (Lesson 0.3). Image-to-image biases the model hard toward your face, but it does not give it a real, lasting identity. Nothing is locked. It is only nudged.
- **It compounds.** Feed an output back in, then feed *its* output back in, over and over, and small errors pile up. It becomes a photocopy of a photocopy, and the character can slowly drift across a long chain.

So the honest summary: image-to-image is the best tool in the cloud box, and it is a **workaround, not a lock.** It manages the gap from Lesson 0.5 better than anything else here, but it does not close it. Truly closing it, teaching the model your character so it carries the identity from the inside, is a different road, and it is exactly what the local path (Level 2) is built for.
