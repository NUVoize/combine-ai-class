# Lesson 1.1: The Three Kinds of Models (Generation, Edit, Hybrid)

In Module 0 we landed on one question that drives everything: *how do I feed visual information into a model that only understands my words as steering?* This lesson is the first real answer, because it turns out the model you choose decides whether you even *can* feed a picture in. There are three kinds, and confusing them is the single most common reason beginners fight their tools.

## 1. Generation models (text to image)

These are the classic ones. You give them a prompt, they start from random noise, and they carve out a brand-new image. That is the whole loop from Lesson 0.1. The only thing going in is your text.

This makes them fantastic at inventing fresh images, and it makes them the "stranger every time" machine from Module 0 when it comes to a specific character. There is no slot to hand them a face. They can only take words, so they can only give you someone who *matches the words*, never necessarily the *same someone*.

## 2. Edit models (image goes in)

This is the first real bridge to everything Module 0 promised. An edit model takes an **existing image as input** and changes it. Inpainting, instructed edits ("put them in a kitchen," "change the jacket to red"), and image-to-image all live here.

Why this matters: visual information finally goes *in*. You are no longer describing a face and hoping. You are handing the model the actual face and asking it to keep that while changing the surroundings. This is the move the whole cloud path is built on.

## 3. Hybrid models (generate AND edit in one)

A hybrid does both jobs in a single model. It can dream up a new image from text, and it can take an image you hand it and edit it, without switching tools. **Nano Banana is the headline example here:** you can generate a character, then in the same place feed that result back in and place them in a new scene, adjust them, iterate.

Hybrids are powerful for character work precisely because they collapse the two moves above into one continuous workflow. Generate once, then keep feeding the result back in, which is exactly the "put the character in front of the model every time" trick from Lesson 0.2, made easy.

## Why this is the first thing in the cloud tier

Tie it straight back to Module 0. Generation-only models are pure text steering, so they live with the full gap we described in Lesson 0.5: words cannot carry a face, so the face wanders. Edit and hybrid models are the answer, because they let you inject the actual visual information that text cannot hold.

So the practical rule for everything that follows: **for a brand-new character or scene, generation is fine. The moment you care about keeping an identity, you reach for edit or hybrid.** Almost all character work in this tier happens in edit and hybrid models, for exactly that reason.
